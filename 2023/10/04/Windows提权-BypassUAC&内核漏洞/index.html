Windows常用的提权技术

Bypass UAC，攻击者可以通过绕过UAC机制来提升进程的权限，UAC（User Account Control）允许进程（通常可以是在从低权限到高权限下运行）以管理员级别的权限来运行，当然这可能会提示用户去确认。

Kernel Exploit，内核漏洞，有些第三方软件也会存在可以提权的漏洞。

Token Impersonation，攻击者可以通过复制并模拟其他用户的token去提升权限并绕过访问控制。攻击者可以创建从现有token中复制来的新的access token，来获取目标用户的权限。

环境

Windows 10，Windows 7，Windows server 2008R2

利用bypassUAC进行Win10提权

首先已经通过msf和empire获取了反弹的shell

使用msf

不选择exploit/windows/local/bypassuac_injection_winsxs模块的原因是这个模块运行在内存中，可以避免一些入侵痕迹以及绕过杀软

运行完之后getuid，用户没有发生改变，但是运行getprivs发现拥有的权限发生了变化，拥有和管理员一样的权限，其中SeImpersonatePrivilege权限是用于模拟access token进行提权的
