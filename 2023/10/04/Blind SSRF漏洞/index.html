Blind SSRF

yu1e
2023-10-04
WebSecurity

What is blind SSRF?

当可以使应用程序向提供的URL发出请求，但后端请求的响应未在应用程序的前端返回，就会出现Blind SSRF漏洞。

What is the impact of blind SSRF vulnerabilities?

由于响应不可见，Blind SSRF 漏洞的影响通常低于响应可见的 SSRF 漏洞。 不能轻易利用它们从后端系统检索敏感数据，尽管在某些情况下可以利用它们来实现完全远程代码执行。

How to find and exploit blind SSRF vulnerabilities

检测Blind SSRF 漏洞的最可靠方法是使用OAST技术（out-of-band HTTP request）。 该技术涉及到尝试触发对已知的外部系统的 HTTP 请求，并监视与该系统的网络交互。

使用OAST技术的最简单和最有效的方法是使用 Burp Collaborator。 可以使用 Burp Collaborator 客户端生成唯一的域名，将这些以有效负载的形式发送到应用程序，并监控与这些域的任何交互。 如果观察到来自应用程序的传入 HTTP 请求，则它容易受到 SSRF 攻击。

在测试 SSRF 漏洞时，通常会观察到对提供的 Collaborator 域名的 DNS 查找，但没有后续的 HTTP 请求。 这通常是因为应用程序试图向该域名发出 HTTP 请求，这导致了 DNS 查找，但实际的 HTTP 请求被防火墙阻止了。 基础设施允许出站 DNS 流量相对常见，但会阻止非预期域名内的 HTTP 连接。

简单地识别一个可以触发带外 HTTP 请求的Blind SSRF 漏洞本身并不能提供利用途径，由于无法查看来自后端请求的响应，因此无法使用该行为来探索应用程序可以访问的内部其他系统上的内容。 但是，它仍然可以用来探测服务器本身或其他后端系统上的其他漏洞，黑客可以盲目扫描内部 IP 地址空间，发送旨在检测公开漏洞的payload。 如果这些payload还采用OAST，那么黑客可能会发现未打补丁的内部服务器上的严重漏洞。

另一种利用Blind SSRF 漏洞的途径是诱导应用程序连接到攻击者控制下的系统，并向建立连接的 HTTP 客户端返回恶意响应。 如果可以利用 HTTP 实现过程中的严重客户端漏洞，就可能能够在应用程序中实现远程代码执行。
